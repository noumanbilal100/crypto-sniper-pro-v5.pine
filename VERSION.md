## Versioning Policy

Use semantic-style version labels for repository releases:

- Patch: documentation, repository metadata, formatting that does not affect strategy behavior.
- Minor: new optional inputs, visuals, dashboard rows, or alerts that preserve existing trading behavior by default.
- Major: changes to entries, exits, signal gates, calculations, risk management, or default strategy behavior.

Any source change must update this file, `RELEASE_NOTES.md`, and `docs/CHANGELOG.md`.
